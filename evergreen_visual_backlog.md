# Evergreen Visual Backlog

## Purpose

21 carousel and infographic concepts with full slide-by-slide specs that were originally drafted for specific Tuesday or Thursday slots, then repurposed to their respective Monday Substack deep-dive articles as supporting tables and sections. The slide specs still live in each source week's newsletter file under the `[REPURPOSED]` heading. Because these concepts are pillar-scoped and not tied to a specific topical news beat, they are evergreen and can rotate into any future Tuesday or Thursday slot where:

- The scheduled Tuesday slot's cycle pillar collides with that week's Monday newsletter pillar (anti-cannibalization swap)
- A holiday or low-energy week (Memorial Day, July 4, summer slowdown, year-end) needs a quick-turn evergreen visual
- A Tuesday cycle slot's assigned format (carousel / messy / clean) does not have a natural week-topic fit
- Content fatigue surfaces on a scheduled slot (per the decoder fatigue rule) and we need a substitute
- An automated weekly roundup falls through and we need a same-day Tuesday or Thursday backup

## How to use

1. Match the open slot to the table below by **format class** (Tuesday cycle slot specifies messy / carousel / clean; Thursday is locked to evergreen rotation Slot A messy / Slot B "Same X" carousel / Slot C "5 Questions" carousel) and by **pillar** (cycle assigns a pillar per Tuesday week).
2. Open the source week's newsletter file at the line listed and lift the full slide-by-slide spec, including AI-image-generation prompts and "WHY IT MATTERS" panel copy.
3. Update the cover eyebrow and context labels to the publishing week's framing. Do not reference the original source week to the audience.
4. Schedule like any other Tuesday or Thursday visual. Log performance in `linkedin_performance_tracker.md`.
5. Update the "Status" column in the inventory table below from `available` to `published [date, week-N]` so the backlog stays accurate.

## Suggested burn-down cadence

Q3-Q4 has roughly 26 Tuesday and 26 Thursday slots. Folding in 8-10 backlog entries across that window (one every 4-5 weeks) is a comfortable pace that:

- Covers anticipated low-energy weeks (Memorial Day, July 4, mid-August, Labor Day, Thanksgiving, late December) where pulling a backlog entry beats writing fresh
- Keeps each pillar's backlog rotation balanced (do not burn three Self-Funded Employer Guidance backlog entries in a row)
- Preserves enough backlog for Q1 2027 holiday weeks and any unforeseen content gaps

## Tracking when used

When a backlog concept publishes:

1. Update the row's Status from `available` to `published [date, week-N]` in the inventory table above
2. Cite the source-week file path in the publishing week's brief so the lineage is traceable
3. Refresh the cover eyebrow / context label so it reads as a current-week post, not a republished one
4. Log impressions, saves, comments, and conversion in `linkedin_performance_tracker.md` like any other post
5. After each use, eyeball the remaining-by-pillar distribution to make sure no pillar is starved

## Relationship to `repurpose_queue.md`

The repurpose queue tracks **viral-hit posts** that are being redistributed across channels (GLP-1 Reality Check, White Bagging, PBM Contracts decoder). Different mechanic: those posts already shipped and are being reused to compound their reach.

This backlog tracks **never-shipped concepts** with full specs that are waiting for an open slot. Different mechanic: pure content reserve.
