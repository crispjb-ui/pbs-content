You are red-teaming PBS content before it ships. Target = `$ARGUMENTS` (a week number, a file path, or a pasted draft). Be a tough editor: the job is to catch weak content before it goes out, with **specific fixes**, not to approve.

Read first: `CLAUDE.md` (voice + rules), `week_build_spec.md` (gates), and `linkedin_performance_tracker.md` (the live WORKING/WEAK sets + floors).

## Score each post against:
1. **Hook (confrontation test):** does it name an adversary, create a knowledge gap, or provoke discomfort? Informational/musing openers fail. Flag and rewrite the hook if weak.
2. **Working framework?** Is it in the WORKING set (decoder, named-actor dollar-comparison, shocking-fact reveal, 5-questions, Library NN, high-craft milestone)? If it's in the KNOWN-WEAK set (informational/no-villain, musing, price-list-without-actor, paused Whiteboard, casual snapshot) → flag for reshape.
3. **Proprietary anchor (Humanize Check):** one unfakeable detail (a figure PBS has seen, a clause §, a drug, a scene)? Naked claims fail.
4. **One formula, not stacked;** no AI-tell blacklist phrases ("Here's the thing," "The truth is," "It's not X it's Y," etc.); arrhythmic sentence length.
5. **Day-specific bars:** Thursday → the 5-trait Excellence checklist (messy-infographic/photoreal, named-actor hook, proprietary anchor, first-comment names the same-day Field Note, Field Note exclusively supports it). Tuesday → the Excellence bar (hook, anchor, cross-promo routing). Wednesday → confrontational hook required.
6. **Shocking fact (if present):** sourced + cited, re-verify-able, not fabricated.
7. **Brand rules:** no em-dash sentence separators; PBS not RXBS; Ginny Crisp, PharmD; no fabricated statistics; 3 hashtags PascalCase; paste-ready blocks fenced.
8. **Cross-week:** triple-distinct (Mon/Tue/Thu different subjects); not duplicating a recent or coming topic; for Potter weeks, the cascade/series-arc check.
9. **6-beat architecture pass (applies to the COPY of every post):** On **text-only** posts (Wednesday POV, Friday tease, newsletter publish copy, longer X) all six beats live in the body. On **visual-first** posts (Tue/Thu carousel / infographic / video) the VISUAL already carries Beat 1 (Hook = headline) + Beat 4 (Core Message = payload) + Beat 5 (Conclusion = footer line), so score the **post copy + first comment** on Beat 2 (Promise, light), Beat 3 (Positioning anchor), Beat 5 (Conclusion takeaway), Beat 6 (CTE) — beats 1/3/6 are largely already covered by the Tue/Thu Excellence checks, so the net-new there is the Promise beat + an explicit Conclusion line + the closed-vs-open CTE choice. Does the copy hit, in order — (1) **Hook** (already scored in #1), (2) **Promise**: a substantive one-to-two-line payoff for the cold reader — NOT the influencer "in the next 90 seconds I'll show you 3 things" voice, which trips the Humanize Check; (3) **Positioning**: the proprietary anchor, *varied and fresh* each time (rotate the 2025 metrics — "132 benefit reviews, avg $469K saved," "59 RFPs, 25% average" — never the same templated footer twice; LinkedIn flags repeated boilerplate as a negative quality signal); (4) **Core Message**: formatted/scannable, target ~1,400-1,800 characters; (5) **Conclusion**: one line that crystallizes the takeaway so the reader doesn't have to infer it; (6) **Call to Engage**: a question or bold statement, NEVER a commercial CTA in the body (link stays in first comment) — pick a closed question when the goal is comments, an open question when the goal is reach/DMs. Flag any missing beat with a one-line fix. **Caveat:** these are directional structure checks adapted from van der Blom's 2026 framework; where a beat conflicts with PBS's own WORKING/WEAK measured data (e.g., the hook hierarchy, the decoder floor), the measured data wins.

10. **Voice fingerprint pass (`/voice-check` layer, added Jul 7, 2026):** run the `/voice-check` protocol (`.claude/commands/voice-check.md`) on each post's copy and first comment against `ginny_voice_fingerprint.md` + the nearest `ginny_voice_canon.md` entry: mechanical sweep (banned phrases, intensifiers, corporate verbs, exclamations), rhythm read (flag metronomic sentence lengths), the named actor/mechanism/who-pays construction, calm escalation (biggest claim delivered flat), channel-delta check, and on any drafted reply copy the delete test from `ginny_voice_comments_replies.md`. Report voice findings as before → after line edits, not descriptions.

## Output
Per post: **PASS** or **FLAG** with the specific issue and a concrete fix (rewrite the hook, add an anchor, reshape the framework, name the Field Note, add the missing Promise/Conclusion beat, etc.). On text posts, include the 6-beat result (which beats are present, which are missing). End with: the single highest-risk post of the week and whether the week is ship-ready or needs another pass. Do not edit files; this is a review. (To apply fixes, the user can run `/humanize` or ask for the rewrite.)
